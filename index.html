Why it matters

A mod-facing surface with engine discipline

Sulkan is not positioned here as a random shader bundle. The source shows a deliberate runtime contract: controlled pack intake, clear quality presets, stable bind names, predictable cascade behavior, and a documentation surface that can support both creators and presentation.

Promotion Built for visual identity

Sun shadows, animated water, and a stronger celestial direction make Sulkan read like a rendering platform with an authored aesthetic, not just a compatibility layer.

Creator workflow Folder packs and zip packs both work

The manager scans the standard `shaders` directory, accepts directory or `.zip` packs, and maps shorthand `shaders/...` layouts into `assets/sulkan/shaders/...` automatically.

Operational model Generated active-pack staging

Selected packs are rebuilt into a dedicated active resource pack so Sulkan can keep pack activation explicit and reload behavior controlled.
