Promotion deck

The rendering layer that makes a Minecraft client feel authored.

Sulkan presents itself best as a premium client-side rendering runtime. The source-backed feature set supports that claim: strong directional shadows, better water, managed shader-pack intake, live preview workflow, and a menu surface built specifically for iterative visual work.

Core promise

Visual control without chaotic pack management

The current implementation shows a consistent theme: Sulkan is opinionated where stability matters, and flexible where creators need room to iterate.

Realtime workflow Swap packs live while staying in-world

The shader UI is designed around in-world preview, immediate selection changes, and explicit reload commands instead of opaque restart loops.

Rendering identity Cascaded terrain shadows with entity integration

Terrain and entity shading are part of one coherent shadow story, giving Sulkan a more cinematic read than a basic post-process stack.

Premium water Displacement, tint, reflection, and refraction

Water is not a flat transparency pass. Sulkan adds motion, thickness-aware tinting, and reflective behavior to make water a headline feature.

PR positioning

How to talk about Sulkan

The messaging below stays ambitious, but still tracks the observed codebase rather than overselling fictional systems.

Short pitch

Sulkan is a Vulkan-first shader runtime for Minecraft that combines live pack switching, managed resource activation, advanced water, and multi-stage shadow rendering into a creator-oriented client experience.

Launch framing

Built for visual modpacks, cinematic setups, technical creators, and players who want a stronger identity than vanilla-compatible shader toggles can usually deliver.

Audience angle

Use Sulkan when visual style is part of the project itself: curated packs, brandable clients, showcase worlds, and shader ecosystems with a defined contract.

Credibility angle

Its strongest differentiator is discipline. Sulkan validates backend assumptions, constrains pack layout, and surfaces quality modes explicitly.

Feature stack

High-signal capabilities

These are the features that matter both technically and for promotion.

Vulkan enforcement

Sulkan forces the preferred graphics API toward Vulkan and rejects non-Vulkan runtime startup.

Shader folder scanning

Supported packs are detected from the game `shaders` directory with unsupported entries clearly ignored.

Quality presets

Low, Normal, High, and Ultra change shadow-map scale, coverage scale, and maximum active cascades.

Live reload

Pack changes trigger resource-pack refreshes and renderer-side invalidation without a full workflow reset.
